# Tetris RL Project

## Goal
Build a reinforcement learning environment and agent that learns to play Tetris, in a single day.

## User Context
- The user cares deeply about **understanding the RL theory** behind every decision — not just getting working code.
- Prioritize explaining **why** over **what**: why this algorithm, why this reward structure, why this network architecture, why this hyperparameter.
- The user wants to understand the **math** — include relevant equations, derivations, and intuitions when introducing concepts.
- Implementation is a vehicle for learning, not the end goal.

## How to Work
- When introducing a new concept (e.g., Q-learning, policy gradients, discount factors), explain the theory and math first, then implement.
- Don't gloss over design decisions — explain trade-offs and alternatives.
- Keep code clean and readable, but don't over-engineer. This is a one-day project.
- When the user asks "why," give a real answer grounded in RL theory, not just "it works better."
